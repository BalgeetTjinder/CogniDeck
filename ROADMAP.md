# CogniDeck — План реализации

## Этап 1: MVP (Minimum Viable Product)

**Цель:** рабочее приложение с базовым циклом — создание карточек, изучение, интервальное повторение.

### 1.1 Инициализация проекта
- [ ] Создать Expo-проект (`expo-router`, TypeScript)
- [ ] Настроить структуру папок (`app/`, `components/`, `lib/`, `assets/`)
- [ ] Подключить `expo-sqlite` и создать схему БД
- [ ] Настроить базовую навигацию (Tab Navigator + Stack)

### 1.2 Схема базы данных

```
decks        (id, title, color, created_at, updated_at)
topics       (id, deck_id, title, created_at)
subtopics    (id, topic_id, title, created_at)
cards        (id, subtopic_id, question, answer, hint,
              easiness, interval, next_review, status, created_at)
```

Статусы карточки: `new` → `learning` → `reviewing` → `mastered` → `archived`

### 1.3 Экраны MVP
- [ ] **Главный экран** — список колод с индикатором (красный/жёлтый/зелёный) и счётчиком карточек на сегодня
- [ ] **Экран колоды** — список тем и подтем, кнопка «Начать сессию»
- [ ] **Создание / редактирование колоды** — название, цвет
- [ ] **Создание / редактирование карточки** — вопрос, ответ, подсказка, привязка к теме/подтеме
- [ ] **Режим изучения** — показ вопроса → показ ответа → оценка (Сложно / Нормально / Легко), прогресс-бар сессии

### 1.4 Алгоритм SM-2 (упрощённый)
- [ ] Реализовать логику пересчёта `easiness`, `interval`, `next_review` после каждой оценки
- [ ] Формирование очереди карточек на день (`next_review <= today`)
- [ ] Начальные параметры: `easiness = 2.5`, `interval = 1`, `next_review = дата создания`

### 1.5 Результат MVP
Пользователь может: создать колоду → добавить темы и карточки → проходить ежедневные сессии → карточки автоматически распределяются по интервалам.

## Этап 2: Визуализация и аналитика

**Цель:** добавить графическое отображение прогресса — граф знаний и статистику.

### 2.1 Граф знаний
- [ ] Подключить `react-native-svg`
- [ ] Реализовать bubble-карту: большие пузыри — темы, маленькие — подтемы
- [ ] Размер пузыря пропорционален количеству карточек
- [ ] Цвет пузыря (красный → жёлтый → зелёный) — средний уровень знания
- [ ] Нажатие на пузырь — краткая статистика + кнопка «Учить только это»
- [ ] Оптимизация: при большом количестве карточек (500+) упрощать визуализацию

### 2.2 Экран статистики
- [ ] Подключить `react-native-gifted-charts`
- [ ] **Серия дней (стрик)** — отображение текущей серии занятий
- [ ] **Тепловая карта активности** — по дням, аналог GitHub contributions
- [ ] **Распределение карточек** — круговая / столбчатая диаграмма по статусам (сложно / нормально / легко / архив)

### 2.3 Таблица `review_history`
- [ ] Добавить таблицу для логирования каждой оценки: `(id, card_id, rating, reviewed_at)`
- [ ] На её основе строить графики и считать стрики

### 2.4 Результат этапа 2
Пользователь видит визуальную карту знаний, понимает где пробелы, отслеживает прогресс по дням.

## Этап 3: Удобство и удержание

**Цель:** функции, которые делают ежедневное использование комфортнее и формируют привычку.

### 3.1 Уведомления
- [ ] Подключить `expo-notifications`
- [ ] Настройка времени ежедневного напоминания
- [ ] Текст уведомления: «У тебя N карточек на сегодня»
- [ ] Экран настроек для управления уведомлениями

### 3.2 Режим «Только сложные»
- [ ] Отдельная сессия из карточек со статусом `learning` или последней оценкой «Сложно»
- [ ] Доступ с экрана колоды и с главного экрана

### 3.3 Поиск
- [ ] Глобальный поиск по тексту карточек (вопрос + ответ)
- [ ] Фильтрация по колоде / теме

### 3.4 Пакетные операции
- [ ] Удаление нескольких карточек
- [ ] Перемещение карточек между темами / подтемами
- [ ] Пакетный импорт карточек из текста (формат: вопрос;ответ на каждой строке)

### 3.5 Тёмная тема
- [ ] Поддержка светлой и тёмной темы (системная настройка + ручной переключатель)

### 3.6 Результат этапа 3
Приложение удобно для ежедневного использования, формирует привычку через уведомления и стрики.

## Этап 4: Финальный продукт

**Цель:** полностью готовый продукт для публикации в сторы.

### 4.1 Экспорт / Импорт
- [ ] Экспорт колоды в JSON-файл через `expo-file-system` + `expo-sharing`
- [ ] Импорт колоды из JSON-файла
- [ ] Валидация структуры при импорте

### 4.2 Онбординг
- [ ] Экран приветствия при первом запуске (3 слайда с пояснением концепции)
- [ ] Пример колоды «Добро пожаловать в CogniDeck» с 5–10 карточками

### 4.3 Полировка UI/UX
- [ ] Анимации переходов между экранами
- [ ] Анимация переворачивания карточки
- [ ] Haptic feedback при оценке карточки
- [ ] Пустые состояния (empty states) — когда нет колод, нет карточек на сегодня
- [ ] Поздравительный экран при завершении сессии

### 4.4 Производительность
- [ ] Профилирование на слабых устройствах
- [ ] Ленивая загрузка списков (FlatList с пагинацией)
- [ ] Оптимизация SQL-запросов (индексы на `next_review`, `deck_id`, `subtopic_id`)

### 4.5 Подготовка к публикации
- [ ] Иконка приложения и splash screen
- [ ] Скриншоты для сторов
- [ ] Описание и метаданные для App Store / Google Play
- [ ] Политика конфиденциальности (данные хранятся только локально)

### 4.6 Результат этапа 4
Приложение готово к публикации: отполированный UI, стабильная работа, все заявленные функции реализованы.
